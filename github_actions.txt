GitHub Actions — це потужний інструмент для автоматизації завдань, таких як тестування, 
збірка та розгортання програмного забезпечення. Давайте розберемося, як він працює.

### Основні компоненти:

1. **Workflow (робочий процес)**:
   - Це автоматизована процедура, яка складається з одного або більше завдань (jobs).
   - Розташований у вашому репозиторії в директорії `.github/workflows`.
   - Написаний у форматі YAML.

2. **Events (події)**:
   - Вказують, коли запускається workflow. Наприклад, при створенні pull request, при коміті у гілку або вручну.
   - Приклади подій: `push`, `pull_request`, `schedule`, `workflow_dispatch` (ручний запуск).

3. **Jobs (завдання)**:
   - Це набір кроків, які виконуються на певному середовищі (наприклад, на Ubuntu або Windows).
   - Можуть виконуватися послідовно або паралельно.

4. **Steps (кроки)**:
   - Окремі кроки всередині завдання, які можуть виконувати команди оболонки або дії (actions).
   - Можуть бути з вбудованими командами або використовувати попередньо створені дії з `GitHub Actions Marketplace`.

### Приклад Workflow:
Розглянемо простий приклад:

```yaml
name: Example Workflow

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '14'

    - name: Install dependencies
      run: npm install

    - name: Run tests
      run: npm test
```

### Пояснення:
1. **name**: Назва робочого процесу - "Example Workflow".
2. **on**: Робочий процес запускається при коміті в гілку `main`.
3. **jobs**: Містить одне завдання `build`.
4. **runs-on**: Завдання виконується на `ubuntu-latest`.
5. **steps**:
   - `Checkout code`: Крок, який використовує попередньо створену дію для завантаження коду репозиторію.
   - `Set up Node.js`: Налаштування Node.js середовища.
   - `Install dependencies`: Встановлення залежностей за допомогою `npm install`.
   - `Run tests`: Запуск тестів за допомогою `npm test`.

Цей приклад показує простий робочий процес для проекту Node.js, який виконується при кожному коміті в гілку `main`.
